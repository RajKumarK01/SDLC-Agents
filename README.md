# SDLC Agent Platform

Model-agnostic, repository-native AI agents for automating the software development lifecycle.

## What This Is

A lightweight platform that adds AI-powered agents to your repositories. Agents analyze your code, review PRs, generate implementations, and write tests — all configured via a simple `.agents/` directory in each repo.

**Current status: POC — Context Builder agent only.**

## Quick Start (5 minutes)

### 1. Clone and build the platform

```bash
git clone https://github.com/YOUR-ORG/sdlc-agents.git
cd sdlc-agents
npm install
npm run build
```

### 2. Set your API key

```bash
export ANTHROPIC_API_KEY=sk-ant-your-key-here
```

### 3. Run the Context Builder on your repo

```bash
# Dry run first (no API call, just scans the repo)
npm run context-builder -- --repo /path/to/your/repo --dry-run

# Full run (calls Claude, generates documentation)
npm run context-builder -- --repo /path/to/your/repo
```

### 4. Check the output

The agent creates three files in your repo:

```
your-repo/.agents/docs/
├── tech-stack.yaml     # Detected languages, frameworks, dependencies
├── architecture.md     # Project structure, patterns, data flow
└── conventions.md      # Naming conventions, code organization, testing patterns
```

## How It Works

### Architecture (4 Layers)

```
┌─────────────────────────────────────────────────────┐
│  Layer 4: TRIGGERS                                  │
│  GitHub Actions, Issue labels, Slack/Teams, CLI     │
├─────────────────────────────────────────────────────┤
│  Layer 3: AGENTS                                    │
│  Context Builder → Code Reviewer → Code Gen → Test  │
├─────────────────────────────────────────────────────┤
│  Layer 2: CONTEXT                                   │
│  Org defaults → Stack profiles → Repo overrides     │
├─────────────────────────────────────────────────────┤
│  Layer 1: MODEL ROUTING                             │
│  Anthropic / OpenAI / Future providers via adapter  │
└─────────────────────────────────────────────────────┘
```

### The `.agents/` Directory

Every participating repo gets a `.agents/` directory:

```
.agents/
├── config.yaml          # Model routing, agent toggles, guardrails
├── context.yaml         # Repo-specific context (domain, patterns)
├── stack-profile.yaml   # Tech stack reference (auto or manual)
├── prompts/             # Model-specific prompt overrides (optional)
└── docs/                # Generated by Context Builder
```

### Layered Context

Context merges in priority order (repo wins):

1. **Org defaults** — Security policies, PR conventions (shared across all repos)
2. **Stack profiles** — Framework-specific conventions (e.g., `dotnet-8-webapi`)
3. **Repo overrides** — This repo's unique patterns, domain, architecture

### Model-Agnostic Design

The platform routes to any AI provider. Currently supports:
- **Anthropic** (Claude) — implemented
- **OpenAI** (GPT) — interface ready, implementation TODO

Swap models by changing one line in `config.yaml`. Prompt templates are per-model for quality.

## CLI Reference

```bash
# Full run
npm run context-builder -- --repo /path/to/repo

# Dry run (scan only, no API call)
npm run context-builder -- --repo /path/to/repo --dry-run

# Custom model
npm run context-builder -- --repo /path/to/repo --model claude-opus-4-6

# With fallback
npm run context-builder -- --repo /path/to/repo --fallback gpt-4o
```

## Setting Up a New Repo

1. Copy `sample-agents-dir/` into your repo as `.agents/`
2. Edit `.agents/config.yaml` — enable the agents you want
3. Edit `.agents/context.yaml` — add repo-specific context (optional)
4. Set `.agents/stack-profile.yaml` — point to a profile or leave as `auto`
5. Copy `.github/workflows/sdlc-agents.yml` into your repo
6. Add `ANTHROPIC_API_KEY` to your repo's GitHub Secrets
7. Run the Context Builder to generate initial docs

## Adding a New Model Provider

1. Add the SDK dependency to `packages/core/package.json`
2. Implement the `call{Provider}()` function in `packages/core/src/model-router.ts`
3. Add model prefix detection to `detectProvider()`
4. Create prompt templates in `prompt-templates/{task}.{provider}.md`
5. Test with `npm run context-builder -- --repo /path/to/repo --model your-model-name`

## Key Design Decisions

- **GitHub Actions over Codespaces** — cheaper, faster startup, sufficient for ephemeral agent tasks
- **Custom build over GitHub Agent HQ** — full control over orchestration, multi-step workflows, and D365 guardrails
- **TypeScript** — GitHub Actions native, known by Node/TS teams, .NET teams just configure YAML
- **Direct Anthropic SDK over LiteLLM (POC)** — minimal dependencies; swap to LiteLLM when adding more providers
- **D365 FO is review-only** — X++ code generation deferred due to quality/safety concerns
